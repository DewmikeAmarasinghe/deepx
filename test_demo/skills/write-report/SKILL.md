# Report writing (this skill is for written deliverables)

Use this skill when the user wants a **written deliverable** such as a report, memo, analysis, comparison, ADR, executive brief, or recommendation document.

This skill focuses only on **how to structure and write the report clearly and professionally**.

---

## Writing quality standards

Good report writing in this domain means:

- **Synthesize, don’t list.** Present conclusions, reasoning, and implications — not disconnected bullet dumps.
- **Clear narrative flow.** Each section should logically lead into the next.
- **Evidence-backed claims.** Any factual or analytical claim must be supported by references, data, or prior findings.
- **Professional tone.** Use clear, neutral, business-appropriate language.
- **Standalone readability.** The document must make sense on its own.

Avoid meta-writing such as:

- “I researched…”
- “I found that…”
- “As an AI…”

Write as if the document is being directly sent to stakeholders.

---

## Required report structure

Use this default structure unless the user requests otherwise:

```text
Executive Summary
Background / Context
Key Findings
Analysis
Recommendations / Next Steps
Sources / References
```

---

## Section guidance

### Executive Summary
A short high-level overview of:
- the core issue
- main findings
- key recommendation

This section should be readable in under one minute.

---

### Background / Context
Briefly explain:
- what problem is being addressed
- relevant scope
- assumptions
- important context

Keep this concise.

---

### Key Findings
Present the most important facts, observations, or results.

Use:
- bullets
- short paragraphs
- tables when useful

Prioritize clarity.

---

### Analysis
This is the reasoning section.

Focus on:
- interpretation
- trade-offs
- comparisons
- implications
- root-cause analysis if relevant

This section should explain **why the findings matter**.

---

### Recommendations / Next Steps
Provide:
- concrete actions
- prioritization
- risks
- follow-up steps

Recommendations should be actionable and decision-oriented.

---

### Sources / References
Always end with a clear references section.

Example:

```text
[1] Report title or source name
[2] Internal document / dataset
[3] URL or citation
```

Keep numbering consistent throughout the document.

---

## Formatting standards

- Use clear Markdown headings
- Keep paragraphs short
- Use bullets for readability
- Use tables for structured comparisons
- Keep tone concise and executive-friendly

The final report should be polished enough to send directly to stakeholders.
